## 1. 農家を口説くロジック

SEN社の経営会議AIチームとして、JAや市場出荷における農家の手取り割合、およびSEN農業ECにおける手取りの変化、その根拠と農家を説得するための比較表、トークスクリプトについて深く分析しました。

### 1. JAや市場出荷における農家の手取り割合の調査結果

農林水産省の調査（2013年度）によると、青果物（調査対象16品目）の**小売価格に占める生産者受取価格の割合は平均で約45.8%**です。残りの約54.2%は集出荷団体経費、卸売経費、仲卸経費、小売経費といった流通経費として差し引かれています。

JA出荷の場合、手数料は売上額の10%〜20%が目安とされています。これにはJA手数料、全農手数料、市場手数料、部会費、選果場利用料、運賃などが含まれます。 ただし、この「売上額」はJAが市場に卸す際の価格であり、最終的な小売価格に対する農家の手取りはさらに低くなります。例えば、ある農家のキュウリ出荷例では、販売金額の約15.3%が手数料として差し引かれ、手元に残る割合が40%程度になったケースも報告されています。

市場外流通（直販など）であれば、生産者の取り分は60%を超えるケースもありますが、一般的な市場流通では30%前後とされます。

## 2. 高単価・まとめ売り設計

# SEN経営会議AI — 「安売りしない農業EC」深度分析

## 前提確認：なぜこの論点が今重要か

現在フェーズ（借金290万・ランウェイ1.4ヶ月）において農業ECは「仕込み期間」だが、**7月頭の撮影ゴールが迫っている**。撮影で何を・どう見せるかが価格設計に直結する。撮影前に「高単価で売る構造」が決まっていないと、撮った素材が「安売り用コンテンツ」になる。今決める必要がある。

**高単価が成立している商品の共通条件:**

4. 送料込みで「損した感がない」価格設定

| 設計 | 農家の実利 | 消費者の心理 |
|-----|---------|-----------|
| 単品少量（安い） | 送料負担率が高く利益率が下がる | 「試し買い」で終わる。リピートしない |
| セット（中〜高単価） | 送料当たりの粗利が改善。1件あたり売上増 | 「お得感」より「まとめると美味しく食べられる」訴求が効く |

# 🟡 発散AI｜「安売りしない」ための設計オプション全展開

## 軸1：価格設計

### A. 「市場の2〜4倍」価格帯の根拠を作る

高単価の正当化は「品質が高い」では弱い。**「なぜこの農家のこの野菜がこの値段か」という文脈**が必要。

**文脈を作る4要素:**

```
1. 農家のストーリー（誰が・なぜ・どう作ったか）
2. 生産方法の差別化（農薬・肥料・栽培環境の具体的数字）
3. 希少性（この農家でしか買えない理由）
4. 体験価値（届いた後に何が起きるか）
```

**価格帯の目安（SENが参入する場合）:**

| カテゴリ | 狙う価格帯 | 根拠となる文脈例 |
|---------|---------|--------------|
| 野菜（旬品） | 市場の3倍 | 「規格外品を出さない。農家が全量チェックして箱詰めする」 |
| 果物（高糖度） | 市場の3〜5倍 | 「糖度◯度以上のみ出荷。それ以外は農家家族が食べる」 |
| 米 | 市場の2〜3倍 | 「収穫後◯日以内に精米。農家が名前を入れて出荷する」 |
| セット（季節便） | 単品合計の1.1〜1.2倍 | 「旬の3種を農家がキュレーションして届ける。毎回違う」 |

### B. 「定価を下げない」ルールを最初に設計する

安売りの起点は「売れないから値下げする」という判断。これを構造的に防ぐ。

**安売り防止ルール（農家との合意事項として契約書に入れる）:**

```
1. 値引き販売禁止（クーポン・セール不可）
2. 「在庫があるから売り切り価格」禁止
3. 価格を下げる代わりに「数量を絞る」
4. 売れ残りは農家が引き取る（消費者に「処分品」を届けない）
```

**値引きの代わりに使う手法:**
- 「今月の農家選び直し便」（同価格で旬の商品に差し替え）
- 「農家訪問権」の付与（高単価購入者に現地見学の招待）
- 送料無料キャンペーン（価格は下げないが「もらえる感」を出す）

# 🔴 否定AI｜穴を全て叩く

## 穴①：送料負担の設計がない → 購入直前離脱が高確率で起きる

**現実**：

ECの購入離脱率の最大原因は「送料が高い」が全業種通じてNo.1（Baymard Institute調査：カート離脱の48%が「予想外の送料」）。

農産品ECは特に深刻。商品1,000円に送料1,800円が乗った瞬間、購入をやめる。

**SENの設計上の問題**：

- 送料を農家負担にすると農家の手取り70%がさらに削れる（実質40〜50%になる）
- 送料を購入者負担にすると離脱率が上がる
- 送料込みにするには商品単価を上げる必要があるが、農産品で高単価化するには「なぜこの農家の野菜が2,000円/kgなのか」の説明が必要

**現在のSEN設計には送料の扱いが一切記載されていない。これは設計の穴ではなく設計の欠如。**

---

## 穴②：農家が発送業務に慣れていない → クオリティのバラつきと発送遅延が起きる

**現実の農家の発送実態**：

- 梱包資材を持っていない農家が多い（段ボール・緩衝材・ネット・保冷剤）
- 宅急便の集荷依頼の手順を知らない農家がいる
- 農作業の合間に「1〜3箱」を梱包して発送する工数が想定以上に重い
- 収穫日に発送できない場合（雨天・農繁期）の代替フローが存在しない

**SENのリスク**：

SENは「注文が来たら農家が直接発送」と書いているが、この「直接」の中に梱包・伝票印刷・集荷依頼・追跡番号の連携・問い合わせ対応が全て入っている。農家に全部やらせると農家が疲弊してEC離脱する。

---

## 穴③：注文集中（バズった瞬間）に対応できない

**ポケットマルシェの事例**：

2022年、農家インフルエンサーのリールがバズって500注文が3日で入った。農家は在庫・梱包資材・発送キャパが20注文/日が限界で、480件をキャンセル。返金・謝罪・低評価が殺到し、農家がプラットフォームを退会した。

**なのかのリールがバズった場合のSENのリスク**：

- バズ発生 → 注文集中 → 農家の発送キャパ超過 → キャンセル多発 → SENのEC・ブランドへの信頼崩壊

- 返金処理・謝罪対応は誰がやるのか（SEN？農家？）が未定義

# 🔵 コンテキストAI｜事実と法的背景を並べる

## 委託形態の選択肢と法的性質

### 選択肢1: 販売委託契約（委託販売）

- 農家 = 委託者。SEN = 受託者（代理販売業者）
- 所有権は売れるまで農家に帰属
- SENは農家の代わりに販売する代理人の立場
- 消費者との売買契約の当事者が「SEN」か「農家」かで責任構造が変わる
- **特定商取引法**: 通信販売として規制対象。返品・返金ポリシーの明示が必要

### 選択肢2: 仕入れ→転売モデル（買取型）

- SENが農家から仕入れ、消費者に販売
- 在庫リスクはSENが持つ
- 産直ドロップシッピングの「在庫を持たない」前提と矛盾するため、現設計では不採用

### 選択肢3: プラットフォーム利用契約

- SENはプラットフォームを提供する事業者
- 農家が出品者として消費者と直接売買契約を結ぶ
- SENは手数料を受け取る「場の提供者」
- 楽天・BASE・メルカリと同じ構造
- **消費者保護責任はどこまでSENが負うか**が契約で争点になりやすい

### 産直ドロップシッピングの法的位置づけ

| 取引形態 | 消費者との売買当事者 | 在庫責任 | 発送責任 |
|---------|------------------|---------|---------|
| 委託販売 | SEN（農家名義の場合もあり） | 農家 | 農家または共同 |
| プラットフォーム型 | 農家 | 農家 | 農家 |
| 買取型 | SEN | SEN | SEN |

> **食品の場合の追加規制**: 食品衛生法・JAS法・食品表示法が適用。産地・使用農薬・添加物の表示義務。ECサイト上の商品ページに記載必須。

# 🟡 発散AI｜論点ごとの推奨案の可能性を広げる

## 論点① 委託形態

**推奨案: プラットフォーム利用契約 + 販売委託の複合型**

農家が「出品者」としてSENのECサイトに出品する構造とし、消費者との売買契約の当事者は農家とする。SENは「プラットフォーム提供 + 集客・マーケティング代行」として30%の手数料を受け取る。

理由:
- SENが売買の当事者にならないため、食品の返品・クレームリスクをSENが直接負わない
- 農家が「自分の店を持った」感覚になり、モチベーション設計として機能する
- SENが拡大した際に「各農家 = テナント」として整理しやすい

## 論点② 取り分割合（30%の内訳明示）

**推奨案: 30%の用途を農家に開示する「透明性型」設計**

農家に「30%の内訳」を見せる:

| SEN内の用途 | 割合（月商に対して） |
|------------|-----------------|
| SNS運営・動画制作 | 12% |
| EC運営・受注管理 | 9% |
| プラットフォーム維持費 | 5% |
| 戦略・ブランディング | 4% |
| **合計** | **30%** |

農家は「手数料を取られている」ではなく「チームを月30%でサブスクしている」感覚になる。

段階設計の導入も有効:
- 月商〜50万: 25%（スタートアップ支援価格）
- 月商50〜200万: 28%
- 月商200万〜: 30%

## 論点③ 発送責任

**推奨案: 農家が梱包・発送を担い、SENは発送フローの標準化を提供する**

具体的な取り決め:
- 梱包資材の基準をSENが設定（ブランド統一のため）

## 1. 特定商取引法（特商法）

### 該当取引類型
**通信販売**に該当（インターネット上での販売）。
訪問販売・電話勧誘・連鎖販売等は非該当。

### ドロップシッピング特有の問題：誰が「販売業者」か

特商法の義務は「販売業者」に課される。

| パターン | 法的位置づけ | 義務の所在 |
|---------|-----------|---------|
| SENが農家の代わりに販売する | SENが販売業者 | SENが全義務を負う |
| 農家が自分で販売・SENは集客代理 | 農家が販売業者 | 農家が全義務を負う |
| SENと農家の共同販売 | 実態判断 | 実態に応じて判断 |

**現状の設計（SEN30%手数料・ECサイトはSEN運営）では、SENが販売業者と判断される可能性が高い。**

### 通信販売の法定表示事項（特商法11条）

ECサイト・LP上に以下を必ず表示しなければならない：

| 表示事項 | 内容 | SENの現状リスク |
|---------|------|--------------|
| 販売業者の氏名・名称 | 法人名・代表者名 | ECデモサイトに記載あるか未確認 |
| 住所 | 実際の事務所住所（バーチャルオフィス可） | ウェブサイト住所変更タスクが未完了 |
| 電話番号 | 問合せ対応できる番号 | 記載があるか未確認 |
| 販売価格（送料含む） | 消費税込み総額表示 | 農家直送の送料負担者が不明確 |
| 代金の支払時期・方法 | いつ・どうやって払うか | 設計未確定 |
| 商品の引渡時期 | いつ届くか | 産直ドロップシッピングは農家出荷タイミング依存 |
| 返品・キャンセル条件 | 返品可否・期間・費用負担 | **食品は原則返品不可だが明記が必要** |
| 不良品対応 | 不良品があった場合の対応 | 農家発送の場合、SENが検品できない |

### 電磁的記録による明示（特商法特有の注意点）

注文確認メール・注文完了画面での法定事項の再掲示が必要。

## 2. 食品表示法

### 適用の根拠

農産物を消費者に販売する行為に全面適用。

### 食品表示基準（平成27年内閣府令第10号）が求める表示

#### 生鮮食品（農産物）の場合

| 表示事項 | 詳細 | リスクポイント |
|---------|------|-------------|
| 名称 | 一般的な名称（トマト・コシヒカリ等） | 品種名・ブランド名だけではNG |
| 原産地 | 都道府県名（または市町村名・農場名） | 農家の所在地を特定して記載 |
| 内容量 | グラム・個数等 | 産直の場合、個体差が大きい農産物の表示方法 |
| 消費期限または賞味期限 | 品質保持可能な期限 | **農家が正確な期限設定をできるか** |
| 保存方法 | 冷蔵・常温等 | 輸送中の保存管理 |
| 栄養成分表示 | 一般的な農産物は免除規定あり | 加工したものは必要 |

#### 加工食品（干し野菜・漬物等）を扱う場合

原材料・食品添加物・アレルゲン・製造者住所等が追加で必要。生鮮より要求事項が格段に増える。

### ECでの食品表示の位置づけ

- 実物の容器・包装に表示するのが原則
- 通信販売の場合は「容器包装への表示に代えて、当該表示事項を記載した書面を添付」または「ウェブサイト上での表示」でよいとされるが、実際の行政指導事例では**実物への表示が最も安全**

**産直ドロップシッピングでは農家が梱包→農家が表示を担う必要がある。SENが設計しても農家が実行しなければ意味がない。農家への表示教育・チェック体制が必要。**

## 4AI壁打ち：契約書の論点整理

### 🔵 コンテキストAI｜実務で必要な条項と現状のギャップ

**SENの事業モデルの特殊性（契約書設計に影響する要素）**

| 要素 | 契約上の論点 |
|------|------------|
| 産直ドロップシッピング | 「在庫を持たない」ため、SENは販売代理人ではなく集客・プラットフォーム提供者として設計する必要がある |
| 農家→消費者直送 | 食品衛生法・産地偽装の責任の所在を明確にしないと農家・SEN双方にリスク |
| SEN手数料30% | 「何に対する30%か」（税込か税抜か・返品時の扱い・送料の含む含まないか）が未整理 |
| なのか・さとなの関与 | 農家との契約はSEN法人と農家の2者間。インターンは第三者として設計 |
| リール→EC導線 | SNS上での農家名・商品の露出方法・許諾範囲が必要 |
| 行政書士未提携 | 補助金サポートを将来追加する場合、この契約書の適用範囲外として別契約を要する |

**農産物ECの法的背景（専門家確認が必要な論点）**

- 食品表示法：産地・原材料表示の責任者は「販売者」。SENが販売者名義を持つ場合はSENが責任を負う
- 特定商取引法：インターネット販売への適用。返品・キャンセルポリシーの記載義務
- 農薬取締法：農薬使用に関する情報開示の義務は農家側だが、ECサイト記載の責任はプラットフォーム運営者にも及ぶ可能性
- 食品衛生法：食品の取り扱い・保管・配送の衛生管理責任の帰属

### 🟡 発散AI｜契約書に入れると差別化になる条項

1. **「SENブランドガイドライン遵守条項」**: なのか・さとなが撮影・SNS発信する際の農家の監修権を明記。「農家が投稿内容を事前確認できる」と記載することで農家の信頼を獲得
2. **「農家側の収益透明化条項」**: 月次で売上・手数料・農家手取りをレポートで開示する義務をSEN側が負う。農家が「30%が何に使われているか」を見える化できる
3. **「段階的手数料条項」**: 月商50万以下は20%・以上は30%という段階制を設けると農家が「まず試せる」設計になる（発散案。採用するかは別途判断）
4. **「優先農家制度条項」**: 一定期間・一定売上を達成した農家には手数料を優遇する特典を設けられる余地を残す

### 🔴 否定AI｜この契約書の穴になりやすいポイント

**穴①：「販売者」の定義が最大の地雷**
消費者からの視点で「誰から買ったか」が不明確な場合、食品事故・産地偽装が起きたときにSENが前面に立たされるリスクがある。「SENはプラットフォーム提供者であり販売者は農家本人」と明記しても、特定商取引法上の「通信販売の販売業者」をSENが担う設計にすると責任が変わる。**ここは専門家確認必須。**

**穴②：返品・腐敗・配送事故の責任分界が未整理**
農産物は腐敗リスクがある。「農家→消費者直送」の場合、配送中の品質劣化はどちらの責任か。農家・SEN・配送業者の3者間の責任分界を契約書に落とさないと、消費者クレームが全てSENに来る。

**穴③：消費者への返金義務がSENに発生しうる**
SENがECサイトを運営し決済を受けている場合、返金・キャンセル対応の窓口はSENになる。農家が返金に応じなかった場合、SENが立替る義務が発生しうる。

**穴④：なのか・さとなが撮影した素材の著作権**
農場での撮影素材の著作権は誰に帰属するか。農家が「うちの農場の写真だから使わせるな」と言った場合の対処が必要。

**穴⑤：契約解除後のSNS投稿の扱い**
契約が解除された農家のリール・投稿はどうするか。残存させると農家から「削除しろ」と言われうる。削除義務の期限・方法を定めないと後でトラブルになる。

---

### 🟢 統合AI｜素案設計方針

否定AIの穴を踏まえ、以下の方針で素案を設計する：

1. SENは「プラットフォーム運営・集客代理業者」として定義し、「販売者」は農家本人とする（食品表示法・特商法上の責任の帰属を明確化するため）
2. 返品・腐敗・配送事故の責任分界を条文に明示する
3. 撮影素材の著作権・使用権を農家とSEN双方が使えるよう許諾する条項を設ける
4. 契約解除後のコンテンツ残存期間を明記する
5. 消費者クレームの一次対応窓口をSENが担うが、農家起因の事故は農家が費用を負担する構造にする
