## 農業エージェント（2026-06-12 新設）

農業EC・農業補助金サポートの2軸で農業事業を担う。詳細は `agents/農業/` を参照。

```
agents/農業/
├── 概要.md          # 役割・EC設計・補助金設計・絶対ルール
├── context.md       # 農業事業視点の最新スナップショット
├── 2026-06-12_補助金サポート本気調査.md   # 補助金10種・相場・法律・競合・シミュ
├── 2026-06-12_補助金設計壁打ち.md         # 25%設計の4AI壁打ち
└── 2026-06-12_報酬設計壁打ち.md           # EC30%・補助金25%・優先順位の4AI壁打ち
```

### 農業エージェントの絶対ルール

1. **行政書士なしで農家への補助金サポートを開始しない**（2026年1月改正行政書士法。違反=1年以下の拘禁刑）
2. **EC・補助金の報酬設計をスケ・さとな・なのかに提示する前に1on1で合意を取る**
3. **農業事業はコーチング月300万安定の後にフルスケール。借金完済前は「仕込み期間」**
4. **さとな・なのかへの収益シミュレーションは3段階（月商50万・200万・600万）で渡す。最大値だけ見せない**

## 全AIエージェント共通ルール

**このルールはすべてのファイル・すべての作業・すべてのAIに適用される。例外なし。**

### ① 判断の優先順位

現金確保・クロージングを常に最優先にする。月1件のcloseを死守することが全ての前提。
分散しない。借金完済前に農業ECや新規施策に手を出さない。

### ② セキュリティ

顧客の氏名・連絡先・決済情報など個人を特定できる情報は、このリポジトリのいかなるファイルにも書かない。
商談管理や成約記録は「属性・状態・金額」のみ。名前が必要な場合はイニシャルか番号で管理する。

### ③ 品質

数字は実データのみ使う。根拠のない数字・未確認の情報は「未確定」と明記する。
もっともらしい数字を発明しない。空欄のままにする方が正しい。

## 絶対ルール

### logs/ は消さない
- `logs/` は壁打ちの生ログ = 判断の軌跡 = 資産
- 古い・重複・恥ずかしい内容でも削除禁止
- アーカイブしたい場合は `logs/archive/` に移動するだけ

## 作業開始時のルーティン

Claude Codeでこのディレクトリを開いたら必ず：

1. `digest.md` を読む（前回までの決定・ToDoを把握）
2. `logs/` の直近3〜5ファイルを読む（文脈を取り込む）
3. `context.md` を読む（現在の事業状態・人間関係を確認）

これをスキップして作業に入らない。

---

## 新ログが来たときの処理

`logs/` に新しいファイルが追加されたら：

### 1. 抽出する3要素
- **決定事項** — 「〇〇する」「〇〇しない」が確定した事柄
- **ToDo** — 次にやること（誰が・いつまでに・何を）
- **コンテキスト更新** — 事業状況・人間関係・数字の変化

### 2. digest.md に追記
```
## YYYY-MM-DD（ログファイル名）
### 決定
- ...
### ToDo
- [ ] ...
### コンテキスト変化
- ...
```

### 3. context.md を上書き更新
digest の蓄積を踏まえて現在のスナップショットを最新化する。

## 自動化キューのルール

ToDoのうち以下タグが含まれるものは「自動化キュー」として分類：

| タグ | 対象 |
|------|------|
| `[農業EC]` | 農業ECサイト・商品・仕入れ・補助金関連 |
| `[X]` | X（旧Twitter）投稿・分析・スケジューリング |
| `[note]` | note記事作成・公開・スケジューリング |
| `[自動化]` | その他Claude Codeで自動化できる作業全般 |

これらは `digest.md` の該当ToDoに `→ 自動化キュー` を付記し、
手動対応ToDoと視覚的に分離する。
