# sen-board 運用ルール

## ディレクトリ構成

```
sen-board/
├── logs/           # 壁打ちの生ログ（日付_タイトル.md形式推奨）
├── context.md      # 現在の事業・人間関係・状態の最新スナップショット（HUBの核）
├── digest.md       # 決定事項・ToDoの累積ログ（時系列追記）
├── CLAUDE.md       # このファイル
└── agents/                    # AIエージェント（7体）
    ├── 全体管理/
    │   ├── 概要.md            # 情報ハブの役割定義
    │   └── context.md         # 全エージェント共通の全体情報
    ├── 営業/
    │   ├── 概要.md            # 役割・商品・課題
    │   ├── マニュアル.md      # 商談フロー・断り返し・フォローアップ
    │   ├── 商談管理.md        # 商談パイプライン（実数入り）
    │   └── context.md         # 営業視点のcontext（パイプライン・成約ルール）
    ├── 財務/
    │   ├── 概要.md            # 役割・監視ルール
    │   └── context.md         # 財務視点のcontext（実数・返済シミュ）
    ├── マーケティング/
    │   ├── 概要.md            # 役割・自動化キュー
    │   └── context.md         # マーケ視点のcontext（集客実態・チャネル）
    ├── プロダクト/
    │   ├── 概要.md            # 役割・ロードマップ
    │   └── context.md         # プロダクト視点のcontext（SENアプリ・優先順位）
    ├── 人材/
    │   ├── 概要.md            # 役割・管理対象
    │   └── context.md         # 人材視点のcontext（各メンバー詳細・リスク）
    └── 法務/
        ├── 概要.md            # 役割・稼働タイミング
        └── context.md         # 法務視点のcontext（リスク・整備すべき契約書）
```

---

## 絶対ルール

### logs/ は消さない
- `logs/` は壁打ちの生ログ = 判断の軌跡 = 資産
- 古い・重複・恥ずかしい内容でも削除禁止
- アーカイブしたい場合は `logs/archive/` に移動するだけ

---

## 作業開始時のルーティン

Claude Codeでこのディレクトリを開いたら必ず：

1. `digest.md` を読む（前回までの決定・ToDoを把握）
2. `logs/` の直近3〜5ファイルを読む（文脈を取り込む）
3. `context.md` を読む（現在の事業状態・人間関係を確認）

これをスキップして作業に入らない。

---

## 新ログが来たときの処理

`logs/` に新しいファイルが追加されたら：

### 1. 抽出する3要素
- **決定事項** — 「〇〇する」「〇〇しない」が確定した事柄
- **ToDo** — 次にやること（誰が・いつまでに・何を）
- **コンテキスト更新** — 事業状況・人間関係・数字の変化

### 2. digest.md に追記
```
## YYYY-MM-DD（ログファイル名）
### 決定
- ...
### ToDo
- [ ] ...
### コンテキスト変化
- ...
```

### 3. context.md を上書き更新
digest の蓄積を踏まえて現在のスナップショットを最新化する。

## 自動化キューのルール

ToDoのうち以下タグが含まれるものは「自動化キュー」として分類：

| タグ | 対象 |
|------|------|
| `[農業EC]` | 農業ECサイト・商品・仕入れ・補助金関連 |
| `[X]` | X（旧Twitter）投稿・分析・スケジューリング |
| `[note]` | note記事作成・公開・スケジューリング |
| `[自動化]` | その他Claude Codeで自動化できる作業全般 |

これらは `digest.md` の該当ToDoに `→ 自動化キュー` を付記し、
手動対応ToDoと視覚的に分離する。

## digest更新コマンド

`/sen-digest` を実行するとlogs/を読んでdigest・contextを更新する。
コマンド定義: `~/.claude/commands/sen-digest.md`
